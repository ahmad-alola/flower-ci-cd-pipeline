# Présentation du projet

## Contexte

Ce projet a été réalisé pour mettre en place un pipeline CI/CD simple autour d’une application de machine learning.

L’objectif principal est de comprendre toutes les étapes du cycle de vie d’un projet :

- données,
- entraînement du modèle,
- exposition du modèle via une API,
- interface utilisateur,
- tests,
- conteneurisation,
- automatisation,
- déploiement.

## Sujet choisi

Pour garder un projet simple et facile à comprendre, j’ai utilisé le dataset **Iris**.

Ce dataset permet de prédire le type d’une fleur à partir de 4 mesures :

- longueur du sépale,
- largeur du sépale,
- longueur du pétale,
- largeur du pétale.

## Pourquoi ce choix ?

J’ai choisi ce dataset car il est simple, connu et suffisant pour montrer un workflow complet.

Le but de l’exercice n’est pas de faire un modèle complexe, mais de mettre en place une chaîne complète et fonctionnelle.

## Résultat attendu

À la fin du projet, on doit avoir :

- un modèle entraîné,
- une API fonctionnelle,
- une interface simple,
- des tests automatisés,
- des conteneurs Docker,
- un pipeline GitHub Actions,
- un déploiement sur Azure.
